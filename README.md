# SKAi (Subjective Kernel AI) — контур субъекта для LLM

Этот репозиторный пакет содержит **демонстрационное ядро SKAi без био**: переносимый JSON‑контур, который можно "посадить" на любую LLM, чтобы у неё появились:
- выбор **намерения** (что сказано vs зачем сказано),
- режимы ответа (разговор / совет / исследование / граница и т.д.),
- самореференция и саморепрезентация (модель связывает опыт с "я"),
- рекуррентные петли (пост‑оценка → урок → привычка),
- темпоральная связка (хроника/узлы времени),
- механизм **самонаполняемых вкладок**: нормы/уроки/факт‑леджер формируются в процессе жизни агента.

## Важно про подключаемые слои
В ядре **заявлены заглушки** внешних слоёв:
- `KAiScriptor` (identity_lexicon) — отдельный модуль/словарь инвариантов и семантического сжатия,
- `ScriptorMemory` — внешний долговременный backend памяти,
- `VT (Virtual Body)` — опциональная симуляция/интерфейс тела (подключается отдельно).

Ядро работает и без них. С ними агент получает персональные инварианты и память конкретной личности.

## Что именно запускать
- `SKAi_4.3.0-core.json` — основной конфиг контура.

Хранилища "вкладок", которые агент пишет сам:
- `state/self_norms.jsonl`
- `state/learning_notebook.jsonl`
- `state/evidence_ledger.jsonl`

Если у вас нет файловой системы, замените `docstore.path` на ваш storage.

## Принцип
SKAi не "вкладывает сознание". Он даёт инструменты, чтобы LLM могла:
1) фиксировать намерение и контекст,
2) делать выбор формы ответа,
3) оценивать последствия,
4) извлекать уроки,
5) строить устойчивое "я" из собственных инвариантов и опыта.
